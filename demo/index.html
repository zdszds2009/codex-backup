Codex Backup Demo

Export. Restore.
Keep context intact.

This demo shows the intended workflow for Codex Backup: package local Codex Desktop conversation state, inspect what is included, and restore it safely into another Windows profile.

Scope
Projects + threads
Safety
Explicit file opt-in
Restore
Backed up first
Step 1
Select a project root or a thread ID.
Step 2
Run the packager to create a zip archive and inspect the manifest.
Step 3
Restore on another machine or test profile, with existing state backed up first.

Workflow

Input

One project root or one thread ID.

  • Local `.codex` profile
  • Optional project files
→

Package

Create a restore-ready zip.

  • SQLite subset
  • Session index
  • Rollout JSONL
→

Restore

Merge into another profile safely.

  • Back up target files
  • Map project roots
  • Restart Codex Desktop
Terminal demo
python demo\auto_demo.py

  --- Step 1/5: Project overview ---

  --- Step 2/5: Clean output ---
  Output  : C:\Users\user\Desktop\codex-demo-output

  --- Step 3/5: Export package ---
  restore_package_zip=codex-backup-demo.zip
  thread_count=15, session_index_entries=15

  --- Step 4/5: Inspect package ---
  Package: codex-backup-demo.zip (568 MB)
  package_name=codex-backup-demo
  thread_count=15, project_count=1
  thread=Codex for Open Source application

  --- Step 5/5: Restore preview ---
  python scripts/restore_package.py --package codex-backup-demo.zip

  Demo Complete!

Run python demo\auto_demo.py on your own machine to generate a real package from your Codex projects. The output above is from a live export of 15 conversations (568 MB) — package inspection, manifest summary, and restore instructions are all included.
