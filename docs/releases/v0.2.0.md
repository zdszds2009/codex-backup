# v0.2.0

Verification and application-readiness release for Codex Backup.

## Summary

This release strengthens Codex Backup as a public maintainer tool by adding end-to-end restore validation, package inspection output, compatibility notes, and cleaner Windows reliability around SQLite file handling.

## Highlights

- Add an end-to-end fixture test that packages a selected thread, restores it into a separate target profile, and verifies path remapping, rollout normalization, and target-state backups
- Add `scripts/inspect_package.py` so maintainers can review package contents before sharing or restoring them
- Record a small source-schema snapshot in `manifest.json` for compatibility debugging
- Fix rollout package-path metadata so the manifest matches the actual archive layout
- Explicitly close SQLite connections to avoid Windows file-lock issues during packaging and restore

## Safety and maintainer value

- Package review is easier before sharing sensitive project state
- Restore behavior is now backed by a public end-to-end fixture, not only unit-level smoke coverage
- Compatibility expectations are documented in `docs/compatibility.md`

## Validation

```powershell
python -m unittest discover -s tests -p "test_*.py"
python -m py_compile .\scripts\build_restore_package.py .\scripts\restore_package.py .\scripts\inspect_package.py .\tests\test_cli_smoke.py
```

## Known limitations

- Windows-focused path handling
- Assumes a compatible local `state_5.sqlite` schema
- No guarantee against future Codex Desktop storage changes
