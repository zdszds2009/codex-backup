# v0.3.0

Community validation and application-readiness release for Codex Backup.

## Summary

This release prepares the repository for legitimate external review before the OpenAI Codex for Open Source application. It adds feedback templates, outreach guidance, and a clear application success plan so reviewers can leave useful public signals without exposing private Codex data.

## Highlights

- Add a usage-feedback issue template for README reviews, `--list` checks, package inspection, and test restores
- Add `docs/outreach-cn.md` with Chinese SMS/WeChat copy, friend testing steps, and GitHub feedback templates
- Add `docs/user-feedback.md` with privacy-safe review paths and maintainer response guidance
- Add `docs/application-success-plan.md` with positioning, evidence targets, and final application answer drafts
- Update the README with a clearer audience section and public feedback request

## Why this matters

OpenAI's Codex for Open Source application is stronger when the repository shows real public value, active maintenance, and evidence from users outside the original author. This release makes it easier to collect honest feedback from real GitHub users while avoiding artificial adoption signals.

## Integrity note

External feedback should come from the reviewer's own GitHub account. Do not post on another person's behalf or create fake usage signals.

- Windows-focused path handling
- Assumes a compatible local `state_5.sqlite` schema
- No guarantee against future Codex Desktop storage changes
