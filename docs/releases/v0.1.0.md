# v0.1.0

First public release candidate for Codex Backup.

## Summary

This release packages local Codex Desktop conversations and optional project files into a portable restore archive for migration and recovery workflows on Windows.

## Highlights

- Export one or more projects by project root
- Export conversations by thread ID or title keyword
- Keep archived conversations in restore packages
- Normalize `model_provider` metadata to improve current Codex Desktop visibility
- Restore into another Windows Codex Desktop profile with automatic backup of local state files

## Safety defaults

- Thread-based exports package conversation data by default without automatically copying cwd project files
- Project files are only included for explicit `--project` exports or with `--include-thread-cwd-files`
- Restore backs up target state files before merge

## Validation

```powershell
python -m unittest discover -s tests -p "test_*.py"
python -m py_compile .\scripts\build_restore_package.py .\scripts\restore_package.py
```

## Known limitations

- Windows-focused path handling
- Assumes a compatible local `state_5.sqlite` schema
- No guarantee against future Codex Desktop storage changes
