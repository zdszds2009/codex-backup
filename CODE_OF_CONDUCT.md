# Code of Conduct

## Our standard

This project is intended to be useful, direct, and safe to collaborate on. Participants are expected to communicate clearly, stay technical, and avoid personal attacks or performative conflict.

## Expected behavior

- Focus feedback on behavior, code, or documentation
- State reproduction steps and observed impact when reporting bugs
- Surface privacy or data-handling concerns early
- Assume mistakes are more likely than malice

## Unacceptable behavior

- Harassment, threats, or discriminatory language
- Publishing private user data from generated restore packages
- Knowingly submitting misleading security reports
- Derailing technical discussions with personal hostility

## Enforcement

Maintainers may remove comments, close discussions, or block contributors who repeatedly violate these standards.
